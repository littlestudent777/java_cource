int a = 120;
1) byte b = a + 10; //error: incompatible types
2) byte c = (byte)(a + 10); //-126
3) byte d = a + 1; //error: incompatible types

1) a + 10 выполняется в контексте типа int, так как a — это переменная типа int.
Значение a + 10 равно 130.
Поскольку 130 превышает максимально допустимое значение для типа byte (которое составляет 127), этот код вызовет ошибку компиляции. 
Значение 130 не помещается в переменную типа byte, и компилятор не может неявно выполнять преобразование. Таким образом, компиляция будет неуспешной.

2) a + 10 снова дает 130.
Результат явным образом приводится к типу byte.
При приведении 130 к типу byte происходит переполнение. 
В Java значение byte представлено 8 битами. 130 в двоичном виде - это 10000010. 
Поскольку в Java все типы - знаковые, старший бит в этом случае интерпретируется как знак. 
Это приведет к тому, что c получит значение 130 - 256 = -126.

3) Переменная a - это int, и с ней выполняется операция сложения с 1 (которая также является int).
Выражение a + 1 будет вычислено и результат будет типа int.
Результат выражения a + 1, который равен 121, будет иметь тип int. 
Оно помещается в переменную типа byte, но Java требует явного приведения типа.
